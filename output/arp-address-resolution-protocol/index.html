ARP (Address Resolution Protocol)

ARP is a network protocol used to find the hardware (MAC) address corresponding to an IP address in a local area network.

Description

Address Resolution Protocol (ARP) operates at the link layer of the Internet Protocol Suite and is used for mapping an Internet Protocol address (IP address) to a machine's physical address (MAC address) within a local network. When a device wants to communicate with another device on the same LAN, it needs to know the MAC address of the destination device. If the sender knows the IP address but not the MAC address, it broadcasts an ARP request packet to all devices in the network. The device with the matching IP address responds with an ARP reply, providing its MAC address. ARP is essential for IPv4 networks, allowing devices to communicate effectively at the data link layer. However, ARP is also susceptible to various security vulnerabilities, such as ARP spoofing, where an attacker sends false ARP messages to associate their MAC address with the IP address of another device, potentially leading to man-in-the-middle attacks or denial of service.

Examples

  • A computer sends an ARP request to determine the MAC address of a server with a known IP address.
  • A network printer responds to an ARP request from a workstation to enable printing.

Additional Information

  • ARP operates primarily in IPv4 networks; IPv6 uses Neighbor Discovery Protocol (NDP) instead.
  • ARP cache stores resolved IP-to-MAC address mappings to reduce the number of ARP requests.
