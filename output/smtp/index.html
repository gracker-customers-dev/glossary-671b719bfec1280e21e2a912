SMTP

Simple Mail Transfer Protocol, a protocol for sending emails.

Description

SMTP, or Simple Mail Transfer Protocol, is a protocol used for sending and receiving email messages between servers. It operates over the Transmission Control Protocol (TCP), typically using port 25, and facilitates the transfer of email from a client to a server or between servers. SMTP is a text-based protocol that takes commands and responses, allowing users to send messages to one or more recipients. While SMTP is primarily used for sending emails, it can also be used in conjunction with other protocols such as IMAP or POP3, which are used for retrieving emails from a server. In the context of network security, SMTP is often targeted by malicious entities to perform attacks such as spam, phishing, and email spoofing. As such, it is essential to implement security measures like SMTP Authentication, encryption (using STARTTLS), and anti-spam filters to protect against these vulnerabilities. Organizations often configure their mail servers to restrict access to trusted networks or employ security appliances to monitor and filter SMTP traffic.

Examples

  • Sending an email from a user's client to the mail server using SMTP.
  • Forwarding an email from one mail server to another mail server using SMTP.

Additional Information

  • SMTP does not provide encryption by default, making it vulnerable to eavesdropping.
  • Common extensions to SMTP include ESMTP (Extended SMTP) which adds additional features.
