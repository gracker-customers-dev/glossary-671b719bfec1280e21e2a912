FTP

File Transfer Protocol, a standard network protocol used to transfer files between a client and a server.

Description

File Transfer Protocol (FTP) is one of the oldest protocols used for transferring files over the Internet. It operates on a client-server model and utilizes a command-response mechanism for communication. FTP can run in two modes: active and passive. In active mode, the client opens a random port for data transfer, while the server connects back to the client. In passive mode, the server opens a port and the client connects to it, which is more firewall-friendly. FTP typically uses TCP port 21 for control commands and port 20 for data transfer, although other ports can be used in passive mode. While FTP is widely used for uploading and downloading files, it is not inherently secure; data, including user credentials, is transmitted in plain text. This vulnerability has led to the development of secure alternatives such as FTPS (FTP Secure) and SFTP (SSH File Transfer Protocol), which encrypt data to protect it during transmission. Despite its security shortcomings, FTP remains popular due to its simplicity and wide adoption for legitimate file transfer needs.

Examples

  • Uploading website files to a web server using an FTP client.
  • Downloading software packages from an FTP server.

Additional Information

  • FTP can be used interactively or through scripts for automation.
  • Many modern FTP clients support secure connections (FTPS/SFTP).
